# 貪食蛇期末專題：完整流程（Windows + GitHub）

本文件整理「從零到完成」的整套流程，包含：
- 安裝與確認工具（PowerShell / Python / Flask / Git）
- 建立 GitHub Repo 並把專案上傳（push）
- 網頁成果如何跑起來（本機）
- 之後如何「連載更新」到 GitHub
-（可選）後端改成 C 語言、前端維持網頁的跑法

> 本專案路徑以我的電腦為例：`C:\Users\TKU\Documents\joycewan\flask-app`

---

## 1) 開 PowerShell

任一方式都可以：
- `Win + X` → Windows PowerShell / Terminal
- 開始功能表輸入 `PowerShell`
- 在檔案總管進到資料夾後，地址列輸入 `powershell` 按 Enter

---

## 2) 安裝 Python（若尚未安裝）

1. 下載：`https://www.python.org/downloads/windows/`
2. 安裝時務必勾選 **Add python.exe to PATH**
3. 打開 PowerShell 驗證：

```powershell
python --version
```

---

## 3) 建立 Flask 專案（後端先用 Flask 跑網頁）

### 3.1 建立專案資料夾

```powershell
cd "C:\Users\TKU\Documents\joycewan"
mkdir flask-app
cd flask-app
```

### 3.2 建立虛擬環境並安裝 Flask

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install flask
```

### 3.3 建立 `app.py`

```powershell
notepad app.py
```

貼上並存檔：

```python
from flask import Flask, render_template

app = Flask(__name__)

@app.get("/")
def home():
    return render_template("index.html")

if __name__ == "__main__":
    app.run(host="127.0.0.1", port=5000, debug=True)
```

### 3.4 建立前端檔案（Flask 會找 templates/static）

```powershell
mkdir templates
mkdir static
notepad .\templates\index.html
notepad .\static\snake.js
```

（`index.html` / `snake.js` 的內容依專案當下版本為準。）

### 3.5 執行並查看網頁成果

```powershell
python .\app.py
```

瀏覽器開：
- `http://127.0.0.1:5000/`

> 看到 `ERR_CONNECTION_REFUSED` 通常表示 Flask 沒有在跑（PowerShell 視窗被關掉或沒看到 Running on...）。

---

## 4) 安裝 Git（用來上傳到 GitHub）

1. 下載安裝：`https://git-scm.com/download/win`
2. 安裝完成後把 PowerShell 全部關掉重開
3. 驗證：

```powershell
git --version
```

---

## 5) 在 GitHub 建立 Repo

1. 進 GitHub，開 `https://github.com/new`
2. 填 repo 名稱（例如 `snake-game`）
3. Public/Private 任選
4. 建議先不要勾 README / .gitignore / License（避免之後衝突）
5. 建立後，複製 HTTPS Repo URL，例如：
   - `https://github.com/singpraise/snake-game.git`

---

## 6) 把本機專案連結到 GitHub（第一次上傳）

### 6.1 進到專案資料夾

```powershell
cd "C:\Users\TKU\Documents\joycewan\flask-app"
```

### 6.2 建立 `.gitignore`（不要把 `.venv` 上傳）

```powershell
@"
.venv/
__pycache__/
*.pyc
.DS_Store

*.exe
*.obj
*.pdb
*.ilk
build/

scores.json
"@ | Set-Content -Encoding UTF8 .\.gitignore
```

### 6.3 初始化 git、設定 remote、把遠端拉下來（避免衝突）

```powershell
git init
git remote remove origin 2>$null
git remote add origin https://github.com/你的帳號/你的repo.git
git pull origin main --allow-unrelated-histories
git branch -M main
```

> 如果 repo 一開始就有 `README.md`，通常需要 `pull --allow-unrelated-histories`。

### 6.4 第一次 commit 可能會卡：請設定作者資訊

如果看到：
- `Author identity unknown`
- `unable to auto-detect email address`

就先設定（只要一次）：

```powershell
git config --global user.name "J Wan"
git config --global user.email "tw.joycewan@gmail.com"
```

### 6.5 add → commit → push（上傳到 GitHub）

```powershell
git add .
git commit -m "add: snake flask web game"
git push -u origin main
```

完成後回 GitHub repo 頁面按 F5，應該會看到 `app.py`、`templates/`、`static/`、`.gitignore`。

## 8)（可選）後端改成 C 語言、前端維持網頁

本專案額外提供 C 語言後端（HTTP Server）：
- `c_backend/server.c`：用 C 提供網頁與 API
- `web/`：前端頁面與 `web/static/snake.js`

### 8.1 需要 `gcc`（MinGW-w64）

若你已安裝 MinGW-w64，可在 PowerShell 驗證：

```powershell
gcc --version
```

### 8.2 編譯與執行 C 後端

在專案根目錄：

```powershell
cd "C:\Users\TKU\Documents\joycewan\flask-app"
gcc -O2 -std=c11 -D_CRT_SECURE_NO_WARNINGS -o .\server.exe .\c_backend\server.c -lws2_32
.\server.exe 8080
```

瀏覽器開：
- `http://127.0.0.1:8080/`

> 這樣就是「後端 C 語言、前端網頁」的成果。

## 9) 常見錯誤快速排除

- **`ERR_CONNECTION_REFUSED`**
  - 伺服器沒有在跑（PowerShell 視窗關掉、或程式啟動失敗）
- **`TemplateNotFound: index.html`（Flask）**
  - `templates/index.html` 沒有存在、或檔名變成 `index.html.txt`
- **PowerShell 打 `@"` 出現 `>>>`**
  - 你進到多行輸入/或誤入 Python 互動模式，先回到 `PS ...>` 再輸入指令
- **`Author identity unknown`（Git commit 失敗）**
  - 先 `git config --global user.name`、`git config --global user.email`
